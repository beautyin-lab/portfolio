# 30 Demo Sites Design Specification

> 각 데모 사이트의 고유한 디자인 컨셉, 레퍼런스, 차별점, 섹션 구성, 애니메이션 정의.
> 동일 아키타입(AT-xx)을 사용하더라도 색상, 히어로 스타일, 레이아웃 밀도, 애니메이션이 다르면 완전히 다른 사이트로 인식됨.

## 1. 강남 밝은미소 치과 (gangnam-smile)

**카테고리**: medical | **아키타입**: AT-03 | **테마 프리셋**: medical-clean

### 1-1. 디자인 컨셉
"프리미엄 강남 클리닉의 신뢰감 있는 의료 매거진 스타일"

### 1-2. 레퍼런스 사이트
- https://www.barunee.com (바른이치과 - 깔끔한 의료 UI)
- https://www.yonseibom.com (연세봄치과 - 프리미엄 치과)

### 1-3. 핵심 디자인 차별점
| 항목 | 사양 |
|------|-----|
| 색상 팔레트 | Primary `#2563EB` (의료 블루), Secondary `#06B6D4` (시안), Accent `#10B981` (민트) |
| 히어로 스타일 | 페이드 슬라이더 - 진료실/미소 이미지 3장 순환, 좌측 정렬 텍스트 + 예약 CTA |
| 레이아웃 밀도 | 넓은 여백, 카드 간격 32px, 섹션 간격 96px - 청결하고 넓은 느낌 |
| 특수 UI 요소 | 진료 과목별 아이콘 카드, Before/After 슬라이더, 의료진 자격증 배지 |
| 애니메이션 스타일 | 부드럽고 신뢰감 - ease-out 300ms, 과하지 않은 페이드 위주 |

### 1-4. 페이지 섹션 구성
1. **Hero** - 페이드 슬라이더, "밝은 미소를 되찾아 드립니다"
2. **About** (소개) - 병원 철학 + 4개 강점 하이라이트 카드
3. **Services** (진료 안내) - 6개 진료과목 아이콘 그리드 [크게]
4. **Team** (의료진) - 원장 프로필 카드 + 약력 [크게]
5. **Gallery** (시설 안내) - 진료실/대기실 사진 그리드
6. **Testimonials** (환자 후기) - 슬라이더
7. **FAQ** - 아코디언
8. **Contact** (오시는 길) - 지도 + 진료시간표

### 1-5. 애니메이션 컨셉
- **히어로**: 5초 간격 크로스페이드, 텍스트 fade-up 500ms
- **스크롤**: 섹션 진입 시 fade-up (stagger 100ms per card)
- **인터랙션**: 서비스 카드 hover 시 subtle scale(1.02) + shadow 강화

6. **FAQ** - 아코디언
7. **Contact** (오시는 길 + 진료시간) [크게]

### 2-5. 애니메이션 컨셉
- **히어로**: 패럴랙스 스크롤 0.3 비율, 텍스트 scale-up 800ms
- **스크롤**: 카드 slide-up with bounce easing (stagger 150ms)
- **인터랙션**: 진료 과목 호버 시 아이콘 rotate(5deg) + 배경색 전환

## 3. 정앤파트너스 법률사무소 (jung-law)

**카테고리**: legal | **아키타입**: AT-03 | **테마 프리셋**: legal-professional

### 3-1. 디자인 컨셉
"대형 로펌의 권위와 전문성 - 다크 네이비 + 골드 서체 강조"

### 3-2. 레퍼런스 사이트
- https://www.kimchang.com (김앤장 법률사무소 - 권위 있는 로펌)
- https://www.bkl.co.kr (법무법인 광장 - 프로페셔널 톤)

### 3-3. 핵심 디자인 차별점
| 항목 | 사양 |
|------|-----|
| 색상 팔레트 | Primary `#1E3A5F` (네이비), Secondary `#B8860B` (다크 골드), Accent `#8B0000` (다크 레드) |
| 히어로 스타일 | 페이드 슬라이더 - 법정/사무실 이미지, 세리프 대형 타이틀, 상담 예약 CTA |
| 레이아웃 밀도 | 넓은 여백, 텍스트 중심, 최소한의 장식 - 권위적 밀도 |
| 특수 UI 요소 | 업무 분야 아코디언, 성공사례 타임라인, 변호사 프로필 상세 |
| 애니메이션 스타일 | 절제된 모션 - ease 200ms, 페이드만 사용, 화려함 배제 |

### 3-4. 페이지 섹션 구성
1. **Hero** - 페이드 슬라이더, "정의를 향한 한결같은 신념"
2. **About** (사무소 소개) - 설립 연혁 + 핵심 가치
3. **Services** (업무 분야) - 민사/형사/기업/부동산 등 분야별 카드 [크게]
4. **Team** (변호사 소개) - 세로형 프로필 + 학력/경력 [크게]
5. **Testimonials** (성공 사례) - 케이스 요약 카드
6. **FAQ** - 법률 상담 관련 FAQ
7. **Contact** (오시는 길 + 상담 안내)

### 3-5. 애니메이션 컨셉
- **히어로**: 8초 간격 느린 크로스페이드, 텍스트 fade-in 600ms
- **스크롤**: 섹션별 fade-in only (slide 없음 - 무게감 유지)
- **인터랙션**: 업무분야 카드 hover 시 border-bottom 골드 라인 등장

## 구현 우선순위

### Phase 1: 테마 프리셋 색상 업데이트
각 사이트의 고유 색상 팔레트를 theme preset에 반영. 현재 카테고리별 2개 프리셋으로는 30개 사이트를 충분히 차별화할 수 없으므로, 사이트별 색상 오버라이드 시스템 필요.

### Phase 2: 히어로 변형 파라미터 확장
같은 히어로 variant(예: fade-slider)를 사용하더라도 다음 파라미터로 차별화:
- `slideInterval`: 슬라이드 전환 속도 (4초~10초)
- `textAlign`: 텍스트 정렬 (left/center/right)
- `overlayOpacity`: 오버레이 투명도 (0.3~0.7)
- `titleFont`: 세리프/산세리프/붓글씨 선택
- `animationStyle`: 전환 애니메이션 종류

### Phase 3: 섹션 순서 커스터마이징
`DemoSiteRenderer`에서 사이트별 섹션 순서와 크기를 config에서 읽어 동적 렌더링:
```typescript
interface SiteConfig {
  // ... existing fields
  sectionOrder?: string[];  // ['hero', 'gallery', 'services', 'about', ...]
  sectionSizes?: Record<string, 'normal' | 'large' | 'hidden'>;
  customSections?: CustomSection[];
}
```

### Phase 4: 업종 특화 섹션 컴포넌트
각 업종에 필요한 특화 섹션을 추가 개발:
- `PricingTable` - 가격 비교표 (피트니스, 스터디카페, 학원)
- `MenuBoard` - 메뉴판 (카페, 레스토랑)
- `ScheduleGrid` - 시간표 (요가, 학원)
- `ListingCards` - 매물 카드 (부동산)
- `TimelineSection` - 프로세스/연혁 (법률, 인테리어, 웨딩)

### Phase 5: 애니메이션 시스템
사이트별 애니메이션 프로필을 정의:
```typescript
interface AnimationProfile {
  scrollReveal: 'fade-up' | 'slide-left' | 'slide-right' | 'scale-up' | 'clip-reveal';
  duration: number;     // 200-1000ms
  stagger: number;      // 50-200ms per item
  easing: string;       // CSS easing function
  heroAnimation: string; // site-specific hero animation
}
```
